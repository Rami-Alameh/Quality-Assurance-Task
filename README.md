# Quality-Assurance-Task
Manual QA assignment (SauceDemo)
Overview
This repository contains a manual QA assignment performed on the SauceDemo web application.

The goal of this assignment is to demonstrate understanding of test planning, test case design, bug reporting, API testing, and product analysis.

Scope
The following areas were tested:
Authentication (valid and invalid login scenarios)
Product inventory and sorting
Cart functionality
Checkout process
Basic UI and usability

Project Structure

TEST_PLAN
Contains the testing strategy, scope, risks, and approach

TEST_CASES.xlsx
Contains 20 test cases covering functional, negative, and UI scenarios

BUG_REPORTS
Contains identified issues with steps to reproduce and evidence

API_TESTS
Contains API testing using a public API

IMPROVEMENTS
Contains suggested improvements for the product

evidence folder
Contains screenshots for reported bugs

API Testing
A public API (https://reqres.in
) was used since the target application does not expose public API endpoints.

The following scenarios were tested:
GET users
POST users
Negative login scenario

Approach
Testing started with core user flows such as login, adding items to cart, and checkout.

Then additional scenarios were covered including negative cases, validation, and UI behavior.

Focus was placed on identifying high-impact issues and keeping the documentation clear and structured.

Time Spent
Approximately 3 to 4 hours

How to Review
Start with TEST_PLAN to understand the testing approach
Review Test Cases.xlsx for coverage
Check BUG_REPORTS along with evidence
Review API_TESTS
Read IMPROVEMENTS
